## 1. 接口描述

接口请求域名： aiart.tencentcloudapi.com 。

混元生图接口，基于混元大模型，根据输入的文本描述快速生成图片。
默认提供1个并发，代表最多能同时处理1个已提交的任务，上一个任务处理完毕后，才能开始处理下一个任务。

默认接口请求频率限制：20次/秒。

## 2. 输入参数

以下请求参数列表仅列出了接口请求参数和部分公共参数，完整公共参数列表见 [公共请求参数](/document/api/1668/88071)。

| 参数名称 | 必选 | 类型 | 描述 |
|---------|---------|---------|---------|
| Action | 是 | String | [公共参数](/document/api/1668/88071)，本接口取值：SubmitTextToImageJob。 |
| Version | 是 | String | [公共参数](/document/api/1668/88071)，本接口取值：2022-12-29。 |
| Region | 是 | String | [公共参数](/document/api/1668/88071)，详见产品支持的 [地域列表](/document/api/1668/88071#.E5.9C.B0.E5.9F.9F.E5.88.97.E8.A1.A8)。 |
| Prompt | 是 | String | <p>文本描述。<br>算法将根据输入的文本智能生成与之相关的图像。<br>不能为空，推荐使用中文。最多可传8192个 utf-8 字符。</p><br/>示例值：雨中, 竹林, 小路 |
| Images.N | 否 | Array of String | <p>参考图，最多三张图。  - Base64 或 Url 。</p><ul><li>单张图片限制：输入图分辨率单边最小50px，最大5000px；图片base64后大小小于6M ；格式支持 jpg、jpeg、png、bmp、tiff、webp。</li></ul><br/>示例值：["https://xxx.jpeg"] |
| Resolution | 否 | String | <p>生成图分辨率<br>单位为 px。<br>格式：&quot;${宽}:${高}&quot;，说明：分辨率的设置和输入是否有参考图（image_urls/images参数）有关：<br>一、文生图（无参考图）<br>默认分辨率：1024:1024；<br>尺寸约束：宽、高均在 [512, 2048] 像素范围内，宽高乘积 ≤ 1024:1024 像素。<br>二、图生图（有参考图）<br>尺寸约束：宽、高均在 [512, 2048] 像素范围内，宽高乘积 ≤ 1024:1024 像素。</p><p>传入尺寸时（输出自适应，不严格遵循传入尺寸）：<br>输入图分辨率分桶与传入尺寸分桶一致时：按输入图长宽比，缩放至接近 1024:1024 面积输出；<br>输入图分辨率分桶与传入尺寸分桶不一致时：从 尺寸列表 中选取最接近传入尺寸的尺寸输出<br>尺寸列表：2048:512、1984:512、1920:512、1856:512、1792:512、<br>1728:512、1664:512、1600:512、1536:512、1472:576、<br>1408:640、1344:704、1280:768、1216:832、1152:896、<br>1088:960、1024:1024、960:1088、896:1152、832:1216、<br>768:1280、704:1344、640:1408、576:1472、512:1536、<br>512:1600、512:1664、512:1728、512:1792、512:1856、<br>512:1920、512:1984、512:2048、768:1024、720:1280、<br>1024:768、1280:720<br>不传入尺寸时：将传入默认值1024:1024。</p><br/>示例值：1024:1024 |
| Seed | 否 | Integer | <p>随机种子，默认随机。<br>不传：随机种子生成。<br>正数：固定种子生成。<br>扩写开启时固定种子不生效，将保持随机。<br>取值范围：1 - 4294967295</p><br/>示例值：148237123 |
| LogoAdd | 否 | Integer | <p>为生成结果图添加显式水印标识的开关，默认为1。<br>1：添加。<br>0：不添加。<br>其他数值：默认按1处理。<br>建议您使用显著标识来提示结果图使用了 AI 绘画技术，是 AI 生成的图片。</p><br/>示例值：1 |
| LogoParam | 否 | [LogoParam](/document/api/1668/88067#LogoParam) | <p>标识内容设置。<br>默认在生成结果图右下角添加“图片由 AI 生成”字样，您可根据自身需要替换为其他的标识图片。</p> |
| Revise | 否 | Integer | <p>是否开启prompt改写，为空时默认开启，改写预计会增加20s左右耗时。<br>0：关闭改写<br>1：开启改写<br>建议默认开启，如果关闭改写，需要调用方自己接改写，否则对生图效果有较大影响，改写方法可以参考：<a href="https://github.com/Tencent-Hunyuan/HunyuanImage-3.0/tree/main/PE">改写</a><br>示例值：1</p><br/>示例值：0 |

## 4. 示例

### 示例1 正常调用场景

#### 输入示例

```
POST / HTTP/1.1
Host: aiart.tencentcloudapi.com
Content-Type: application/json
X-TC-Action: SubmitTextToImageJob
<公共请求参数>

{
    "Prompt": "生成一张配图**一个日历显示‘Monday’，旁边有一个闹钟指向7:00，窗外阳光明媚代表早晨，书包放在桌边。画面无文字。**；\\n风格要求：**真实风格**，色彩鲜亮、温暖、饱和度高。\\n画面细节：线条简洁流畅，背景柔和，聚焦主体。\\n视觉效果：摄影风，适合儿童教学用途。\\n\\n要求点：\\n- 如果涉及人物的话，人物形象积极健康，充满活力和阳光感；\\n- 图片中的场景、人物都必须是真实世界的事物，不要出现幻想、想象、虚拟等非真实世界的事物；\\n- 图片不得包含任何文本（数字、单词、符号、句子等），如果上面提示词有的话，也要去掉；\\n- 图片内容必须精准匹配描述，避免错误元素（如逻辑不符，图片表示跳出箱子、正确项却为跳进箱子）；\\n- 生成注意符合生活常识（如太阳应出现在室外，而不是房间里面，如大象出现在野外，不在桌子上）；\\n- 涉及到钟表显示，使用电子表，不要使用指针钟表；\\n- 图片中的元素比较精简，不要堆砌多个元素、模糊重点；",
    "Resolution": "1248:832",
    "Revise": 0
}
```

#### 输出示例

```json
{
    "Response": {
        "JobId": "251197765-1769741457-80b5b8df-fd86-11f0-ad04-525400647112-0",
        "RequestId": "ee8f21db-e724-4333-8130-e0520ac73e4f"
    }
}
```
