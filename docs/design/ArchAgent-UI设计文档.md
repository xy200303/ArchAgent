# ArchAgent UI 设计文档

## 1. 目标布局

ArchAgent 是桌面 3D 工作台，而非“聊天旁边放一个预览”。主界面采用三栏：左侧管理项目与场景，中间完成编辑，右侧在属性与 Agent 对话之间切换。

```text
+--------------------------------------------------------------------------------+
| 项目 / 场景树 | 工具栏、视图控制、撤销重做              | 属性 / Agent 页签 |
|               |------------------------------------------|-------------------|
| 资源与材质    |                                          | 属性检查器        |
|               |               3D Viewport                | 或会话消息流      |
|               |                                          |                   |
|               |------------------------------------------|-------------------|
|               | 状态提示、坐标、选择信息                 | 对话输入区        |
+--------------------------------------------------------------------------------+
```

## 1.1 编辑器初始框架

首个可交付的编辑器框架先保证空间、工具和状态的秩序，再按能力上架建筑构件。它不是默认常驻的参数表单。

- **工具栏**：左侧展示文档和当前楼层；中间是选择与当前可用的建模工具；右侧是视图预设和显式参数创建。图标工具必须有 pressed 状态、tooltip、键盘取消路径。
- **工作台侧栏**：54px 活动栏提供独立“构件库”入口；打开后以第二侧栏展示模型组件，并暂时让出编辑器内的场景树宽度。构件库仅展示已经具备命令、渲染和检查器支持的对象，禁止上架占位按钮。
- **编辑器左栏**：场景树用于选择、定位和层级浏览；构件库关闭时显示，避免在狭窄视口同时挤占构件库、场景树与属性面板。
- **中心视口**：始终是主要空间。未选中对象时不显示右侧检查器；视图预设、相机重置、网格与坐标等属于视口控制而不是属性面板。
- **右侧检查器**：仅在用户选择对象或显式进入参数创建时出现。对象类型决定其字段和可用操作，不能默认显示“新建墙体”。
- **首批构件方向**：结构构件为 Wall、Slab、Column；室内构件从 Table、Chair、Sofa、Cabinet、Plant 等可参数化 Item 开始。Door、Window 要等墙洞约束、吸附和校验完成后上架。

视口必须提供透视、顶视、正视、右视与重置相机。相机能力使用 R3F 实现，独立于 Pascal 是否可用。

初始框架的左栏宽度为 172-220px；右侧检查器宽度为 244-300px，且只在选择或参数创建时出现。3D 视口始终拥有剩余空间。场景 JSON/JSCAD 编辑器为底部可折叠面板，不抢占首屏。

## 2. 左栏

- **项目文件**：`input/`、`assets/`、`output/` 与场景 JSON。
- **场景树**：Pascal 层级与 MeshAsset，点击后同步选择。
- **资产浏览器**：家具、静态人物、装饰和材质预设；拖入视口会激活 Asset 工具。

## 3. 中间编辑器

工具栏分为选择/变换、建筑创建、资产、视图和历史五组。工具启用后，视口底部持续显示下一步提示、快捷键和取消方式。

R3F 主视口始终提供相机操作：右键旋转，左键拖动平移，滚轮缩放。点击构件同步场景树和属性检查器；拖动期间不触发场景选择。Pascal 预览不承担相机交互，因此不可用时不会阻断编辑。

视口叠加层包含网格开关、坐标轴、视图预设、WebGPU/WebGL 状态与渲染故障降级入口。

## 4. 右栏

右栏使用“属性 / Agent”页签，避免两个固定面板同时压缩视口。

- **属性**：按 Wall、Door、Window、Slab、Item 或 MeshAsset 展示字段；修改即时预览，在提交后写入命令历史。
- **Agent**：展示计划、工具调用、ghost 预览、校验错误和确认/取消操作。工具调用显示受影响节点和 revision，不展示无关内部日志。

## 5. 状态与反馈

| 状态 | 反馈 |
| --- | --- |
| Agent 规划 | 简短步骤和待确认的变更数量 |
| Ghost 预览 | 半透明对象 + 确认、修改、取消 |
| 命令冲突 | 提示场景已更新，提供重新应用 |
| WebGPU 不可用 | 自动切换 WebGL，保留重试入口 |
| 导入资产 | 进度、格式、尺寸和可放置状态 |

颜色不能作为唯一状态标识；图标按钮提供文本标签或 tooltip，键盘操作提供可见提示。
