# ArchAgent UI 设计文档

## 1. 目标布局

ArchAgent 是桌面建筑 3D 工作台，而非“聊天旁边放一个预览”。主界面采用三栏：左侧管理项目与建筑场景，中间完成编辑，右侧在属性与 Agent 对话之间切换。

```text
+--------------------------------------------------------------------------------+
| 项目 / 场景树 | 工具栏、视图控制、撤销重做              | 属性 / Agent 页签 |
|               |------------------------------------------|-------------------|
| 建筑构件库    |                                          | 属性检查器        |
|               |          Pascal 建筑编辑视图             | 或会话消息流      |
|               |                                          |                   |
|               |------------------------------------------|-------------------|
|               | 状态提示、坐标、选择信息                 | 对话输入区        |
+--------------------------------------------------------------------------------+
```

## 1.1 编辑器初始框架

首个可交付的编辑器框架先保证空间、工具和状态的秩序，再按能力上架建筑构件。它不是默认常驻的参数表单。

- **工具栏**：左侧展示文档和当前楼层；中间是选择与当前可用的建模工具；右侧是视图预设和显式参数创建。图标工具必须有 pressed 状态、tooltip、键盘取消路径。
- **工作台侧栏**：54px 活动栏提供“主工作区”和“构件库”入口。打开构件库后以第二侧栏展示 Pascal 可表达的建筑组件，并暂时让出编辑器内的场景树宽度；禁止上架占位按钮。
- **编辑器左栏**：场景树用于选择、定位和层级浏览；构件库关闭时显示，避免在狭窄视口同时挤占构件库、场景树与属性面板。
- **中心视口**：Pascal 是唯一建筑编辑视图，始终占据主要空间。未选中对象时不显示右侧检查器；视图预设、相机重置、网格与坐标属于视口控制而不是属性面板。
- **右侧检查器**：仅在用户选择对象或显式进入参数创建时出现。对象类型决定其字段和可用操作，不能默认显示“新建墙体”。
- **首批构件方向**：Wall、Slab、Column；Door、Window 要等墙洞约束、吸附和校验完成后上架，随后扩展 Zone、Ceiling、Roof 和建筑 Item。

视口必须提供自由、顶视、正视、右视与重置相机。相机使用嵌入 Pascal Viewer 的 `CameraControls` 实现，左键旋转、右键平移、滚轮缩放。

初始框架的左栏宽度为 172-220px；右侧检查器宽度为 244-300px，且只在选择或参数创建时出现。3D 视口始终拥有剩余空间。场景 JSON/JSCAD 编辑器为底部可折叠面板，不抢占首屏。

## 2. 左栏

- **项目文件**：`input/`、`output/` 与场景 JSON。
- **场景树**：Pascal 建筑层级，点击后同步选择。
- **构件库**：按结构、围护、开口、空间和材质预设分组；仅展示已有命令、Pascal 渲染和检查器支持的建筑组件。

## 3. 中间编辑器

工具栏分为选择、建筑创建、视图和历史四组。工具启用后，视口底部持续显示下一步提示、快捷键和取消方式。

Pascal 主视口提供相机操作：左键旋转，右键平移，滚轮缩放。点击构件同步场景树和属性检查器；拖动相机时不触发场景选择。Pascal 的内置场景承担建筑几何、选择和楼层能力。

视口叠加层包含网格开关、坐标轴、视图预设与 WebGPU 状态。WebGPU 初始化失败时在视口内显示局部错误和“重新加载 Pascal 预览”入口；不回退到第二个 WebGL/R3F 编辑器。

## 4. 右栏

右栏使用“属性 / Agent”页签，避免两个固定面板同时压缩视口。

- **属性**：按 Site、Building、Level、Wall、Door、Window、Slab、Zone 或建筑 Item 展示字段；修改即时预览，在提交后写入命令历史。
- **Agent**：展示计划、工具调用、ghost 预览、校验错误和确认/取消操作。工具调用显示受影响节点和 revision，不展示无关内部日志。

## 5. 状态与反馈

| 状态 | 反馈 |
| --- | --- |
| Agent 规划 | 简短步骤和待确认的变更数量 |
| Ghost 预览 | 半透明建筑构件 + 确认、修改、取消 |
| 命令冲突 | 提示场景已更新，提供重新应用 |
| WebGPU 不可用 | 视口内局部错误与 Pascal 重试入口 |
| 构件不支持 | 说明当前建筑节点能力边界，避免创建不可渲染数据 |

颜色不能作为唯一状态标识；图标按钮提供文本标签或 tooltip，键盘操作提供可见提示。
