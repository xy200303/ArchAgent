# 场景命令 P0

## 目标

让用户和 Agent 通过同一组可验证的命令编辑当前 3D 场景，并在 Pascal 视口中即时同步。

## 本轮范围

- 共享场景模型、默认建筑场景和纯命令 reducer。
- `create_wall`、`update_wall`、`delete_node` 三类命令及其输入校验。
- Main 内存场景服务、IPC 和 Renderer 事件同步。
- Renderer 场景树、选中墙属性编辑和新增墙表单。
- 将三项命令暴露为 Agent 工具。

## 非目标

- 场景项目持久化、门窗、Slab/Zone、撤销重做、Ghost preview、真正的鼠标画墙和变换 gizmo。
- 此阶段场景仅保留在应用运行内存中；重新启动后恢复默认场景。

## 验收

1. 用户可在编辑器内新增、选择、修改和删除墙体。
2. Agent 工具调用与用户操作使用相同命令、校验和事件。
3. 无效 ID、重复 ID、零长度墙和无效尺寸会返回可理解错误，且不改变场景。
4. 新命令在已打开视口中即时生效；重新载入视口可取得当前内存场景快照。

## 最终行为契约

- 默认行为：应用启动时创建一套内存中的建筑场景，包含场地、建筑、楼层、楼板和四面默认墙；场景 revision 从 `0` 开始。
- 正常路径：Renderer 和 Agent 都将墙体新增、修改、删除转为同一 `SceneCommand`，由 Main 服务校验、更新 revision，并向所有 Renderer 广播接受的命令和快照。
- 失败路径：无效父节点、重复或非法 ID、零长度墙、无效尺寸或材质、找不到节点和不支持的节点类型会返回可读错误，不变更快照且不广播事件。
- 交互路径：场景树可选择墙体；右侧表单可新增、修改和删除；默认墙高 `2.8` 符合浏览器输入约束，能够直接提交。
- 生命周期：场景只存于当前应用运行内存；重新启动后恢复默认场景。
- 验证：已通过 TypeScript 类型检查、单元测试、生产构建，以及使用预加载桥 mock 的浏览器交互冒烟测试（新增、修改、删除；1280px 和 900px 布局）。浏览器环境不支持 Pascal 所需的 WebGPU 时，界面会降级为 WebGL 预览并保留场景命令表单。
