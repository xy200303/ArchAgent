# 场景视口 P1

## 目标

让 R3F 成为始终可用的主编辑视口，先建立可靠的相机与选择交互，再叠加画墙、吸附和变换工具。

## 本轮范围

- 从共享 `SceneSnapshot` 渲染楼板和墙体，不读取或修改 Pascal store。
- 提供轨道、平移和缩放；WebGPU 不可用不会阻断视口。
- 点击墙体后同步场景树和属性面板，并提供明确高亮。
- 抽离坐标到 Three.js Box transform 的纯函数并覆盖单元测试。
- 提供画墙工具、0.25 米网格吸附和半透明预览；第二次点击才提交 `wall.create`。

## 非目标

- 墙体端点拖动、gizmo、撤销重做和多选。
- Pascal 相机控制或将 Pascal store 纳入编辑链路。

## 验收

1. 任何支持 WebGL 的运行环境都能显示默认场景。
2. 右键旋转、左键拖动平移、滚轮缩放不会改变场景快照。
3. 点击墙体会选中并高亮，点击空白处取消选择。
4. 场景树、表单和 Agent 命令写入后，R3F 视图由最新 snapshot 更新。
5. 画墙工具的第一次点击只保存临时起点，移动时显示预览，第二次有效点击提交同一条 `wall.create`；按 Escape 或取消工具不修改场景。
