# 专业编辑器初始框架

## 目标

在继续增加建筑构件前，建立一个以视口为中心、工具位置稳定、检查器按需出现、并可扩展构件库的桌面 3D 编辑器框架。

## 本轮范围

- 修复工具组错误纵向排列的问题，重组文档、工具、视图和参数创建入口。
- 视口支持透视、顶视、正视、右视和重置相机等真实相机动作。
- 构件库作为工作台活动栏的独立入口，在第二侧栏展示；打开时编辑器左侧场景树让出空间。检查器只在选择或创建流程中占据空间。
- 现有的选择、画墙、参数创建、墙体修改与删除保留为真实可用能力。

## 后续构件准入

任何内置构件进入构件库前必须满足：

1. 有受 `SceneService` 校验的创建或修改命令。
2. 能从 `SceneSnapshot` 渲染，并能在场景树中选择。
3. 至少有可读检查器；可编辑属性必须走同一命令链路。

## 非目标

- 用不可点击或没有场景命令的图标伪装完整对象库。
- 在没有墙洞关系校验前提供 Door 或 Window 创建。
- 在没有历史服务前显示不可用的撤销、重做工具。

## 验收

1. 工具栏、视图控制和参数创建在桌面与窄宽度下均不重叠、不换成无意义的纵列。
2. 无选中或创建状态时，视口获得检查器原本占用的空间。
3. 相机预设会真实改变 R3F 相机和控制器 target，不写入场景命令。
4. 选择、画墙、参数创建、编辑与删除墙体保持可用。
